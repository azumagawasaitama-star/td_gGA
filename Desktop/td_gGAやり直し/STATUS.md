# TD-gGA 実装状況メモ (2026-05-14)

---

## 論文の要点（Guerci, Capone, Lanata, PRR 5, L032023 (2023)）

### 手法の概要

Ghost-Gutzwiller 近似（gGA）を時間依存に拡張したもの。
通常の GA にゴースト軌道（B−1 本）を追加し、強相関を系統的に改善する。

- 物理電子 1 軌道 ＋ ゴースト B−1 軌道 → 合計 B 個の準軌道（空間）
  - 論文では B=1, 3, 5, 7 を用いる（本実装では B=3、つまりゴースト 2 本）
- 変分状態：|ψ⟩ = Proj × |Ψ₀⟩ × |Φ⟩
  - |Ψ₀⟩：準粒子スレーター行列式（n(ω) で特徴付け）
  - |Φ⟩：埋め込みハミルトニアン H_emb の波動関数

### 時間発展の核心方程式（論文 Eqs. 15–21）

Lagrangian を「**Λ=0 に設定**」することで、以下の TDVP 方程式系が得られる：

| 式番号 | 内容 |
|--------|------|
| Eq.16 | d\|Φ⟩/dt = −i H_emb(D, Λ^c) \|Φ⟩ |
| Eq.17 | dn(ω)/dt = −iω [n, RR†]（**Λ 項なし**）|
| Eq.5  | Δ = ∫ρ(ω) n(ω) dω |
| Eq.18 | D = [Δ(1−Δ)]^{-1/2} ∫ρ(ω)ω n(ω) dω |
| Eq.20 | R = [Δ(1−Δ)]^{-1/2} ⟨f†c⟩_Φ |
| Eq.19 | Λ^c = −Λ − deriv(DR, Δ) （Λ=0 なら後半項のみ）|

**重要**: 「Λ=0」は ODE だけでなく、**静的な自己無撞着計算にも適用される**。
つまり平衡状態も n_eq(ω) = f(ωRR†) で求める（Λ を最適化変数にしない）。

TDVP 構造により、初期条件が拘束条件（F2=0）を満たせばエネルギーは自動的に保存される。

## Route B（Λ≠0 固定）— 頓挫した方針

### 実装内容（`td_gGA_solver_full.py`）

静的計算で求めた Λ（固有値 ±0.67〜±0.75）を時間発展中も定数として固定。

| 式 | 内容 |
|----|------|
| 初期状態 | n₀(ω) = f(ωRR† + Λ) |
| n の ODE | dn/dt = −iω[n, RR†] + i[Λ, n] |
| H_emb の Λ^c | Lc = calc_Lmbdac(Λ, Δ) = Lc_fast − Λ |

### 頓挫した理由

**静的計算と時間発展で保存量の定義が食い違う。**

静的ソルバー（`ga_mainfin.py`）は Λ を自由変数として最適化し、
U=0.05 で Λ≠0（固有値 ±0.67）、U=1.0 で ±0.75 の解を返す。
これは論文の Λ=0 ゲージとは別のゲージであり、Route B の「保存量」は

E_B = ⟨Φ|H_emb(D, Lc_fast − Λ)|Φ⟩

として定義される。しかし Λ を定数に固定したまま |Φ⟩ と n(ω) を
時間発展させると、**この E_B は TDVP 的に保存量ではない**。
変分微分 δE_B/δn = ρ(ω)(ωRR†+Λ) として ODE を作ると、
実際には Λ が n(ω) の時間発展に合わせて変化する必要があるが、
それを無視して固定してしまっている。

### 実行結果（U=1.0→2.5 クエンチ）

| 量 | 値 | 評価 |
|----|----|----|
| Z[0] | 0.763 | ✓ 静的値と一致 |
| E_phys 最大相対誤差 | **64%** | ✗ 完全に保存されない |
| F2 最大値 | 3.6% | ✗ 拘束条件が徐々に崩れる |

dt を半分にしても誤差がほぼ変わらなかった（数値精度の問題ではない）ことも確認済み。

## Route A（Λ=0 ゲージ）— 現在の実装、動作中

### 実装内容（`routeA/` フォルダ）

論文の Λ=0 ゲージを徹底した実装。
静的計算・SC ループ・ODE の**すべてで Λ=0 を強制**する。

**新規ファイル構成（`routeA/` フォルダ）**

| ファイル | 役割 |
|---------|------|
| `ga_mainfin_routeA.py` | `ga_mainfin.py` のコピー＋`optimize_selfc_routeA` 追加 |
| `td_gGA_solver_routeA.py` | SC ループが純粋 Λ=0 版（Route B 混入なし） |
| `test_noquench_routeA.py` | ノークエンチ診断スクリプト |
| `run_paper_quench.py` | 論文 Fig.2 再現スクリプト |

**`optimize_selfc_routeA`（新メソッド）の動作**

1. `self.Lmbda = 0` に固定（全反復で強制）
2. Δ = ∫ρ(ω) f(ωRR†) dω を計算（Λ 項なし）
3. D, Λ^c を代数的に計算
4. H_emb を対角化 → R を更新
5. Λ の最適化ステップ（`cost_func_Delta`）を**完全に省略**
6. R の収束のみで判定（通常 3〜5 反復で収束）

### うまくいっていること

| テスト | 結果 |
|--------|------|
| ノークエンチ d(t) 変動 | **5.5e-11**（機械精度） |
| ノークエンチ E_phys 誤差 | **4.5e-10**（機械精度） |
| ノークエンチ E_emb 誤差 | 1.6e-8（前回 Route B 混入版の 100% から激減）|
| クエンチ E_phys 誤差（最大） | **~10^{-4}**（Route B の 64% から激減）|
| 静的値との一致 | Z[0], d[0] ともに一致 |
| Λ 固有値 | [0, 0, 0] ✓ |

**クエンチ結果（U_i=0.05 → 各 U_f, t_max=5.0）**

| U_f | E_phys 誤差 |
|-----|------------|
| 0.5 | 1.6e-6 |
| 1.0 | 1.9e-5 |
| 1.5 | 1.2e-4 |
| 2.0 | 8.4e-5 |
| 2.5 | 5.1e-5 |

d(t) の定性的挙動（U_f 増大で二重占有が減少、振動）は論文 Fig.2 と一致。

### 残っている問題点

**F2 拘束条件の違反（F2 ≈ 0.47 → 0.71）**

- 静的初期値の時点で F2=0.47（収束判定を通過できない）
- 時間発展中も F2≈0.71 で一定（増大はしないが減少もしない）

**原因:** U=0.05 の弱相関極限で Λ=0 にすると：
- RR† ≈ diag(1, 0, 0)（ゴースト軌道の寄与 ≈ 0）
- f(ωRR†) の縮退部分空間でゴースト軌道は f(0)=0.5 → Δ≈0.5·I
- H_emb の gap = 0（縮退）
- numpy の eigh が縮退部分空間から任意の状態を選んでしまう
- → ⟨f†f⟩ ≠ Δ という F2 違反が生じる

エネルギー保存と d(t) の時間発展自体は正しく動いているため、
**物理的な定性挙動の再現には成功している**が、
定量的な精度（特に初期の d 値の絶対値）には不確かさが残る。

## Route C（動的 Λ 更新）— 次の実装目標

### 基本アイデア

Λ を定数に固定せず、**各時刻に拘束条件から代数的に決定する**。

F2 = ⟨f†f⟩_Φ − Δ(Λ, n) = 0

を Λ について各ステップで解くことで、
制約条件を時間発展中に常に満たし続ける。

### 論文との対応

Route C は論文の「完全な TDVP 方程式系」に相当する。
論文は Λ=0 ゲージを採用して方程式を簡略化しているが、
Λ を動的に決める Route C は（ゲージ自由度があるため）
原理的に同等の物理を与えるはずである。

### 実装方針

各時刻ステップの ODE 右辺を計算する際に：

1. 現在の (|Φ⟩, n(ω)) から Δ, R, D, Λ^c を計算（Route A と同じ）
2. 拘束条件 F2 = ⟨f†f⟩_Φ − Δ(Λ) = 0 を Λ について解く
   - Δ(Λ) = ∫ρ(ω) f(ωRR†+Λ) dω は Λ の非線形関数
   - 現在の `cost_func_Delta`（Levenberg-Marquardt）をそのまま流用できる
3. 求まった Λ を使って ODE の右辺を計算

**利点:** F2≈0 が常に保証される → d(t) などの物理量が定量的に正確になる

**懸念点:**
- 各ステップで最適化ループが走るため**計算コストが大きく増加**
- ODE のヤコビアンが陰的になるため、積分器の選択（Radau, BDF 等）を検討
- Λ の連続性（時間ステップ間で解が飛ばないか）の保証

### 具体的な実装ステップ

1. `td_gGA_solver_routeC.py` を作成（Route B の `td_gGA_solver_full.py` をベースに）
2. `compute_RDLc` 内で、Λ を固定せずに `cost_func_Delta` 相当の最適化を実行
3. `compute_derivatives_full` の ODE 右辺で動的な Λ を使用
4. ノークエンチテストで F2 の収束を確認してからクエンチへ
